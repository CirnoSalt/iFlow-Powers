# Requirements: Superpowers for iFlow CLI

## v1 Requirements

### Core Skills
- [ ] **CORE-01**: Test-Driven Development (TDD) 技能
  - 强制 RED-GREEN-REFACTOR 循环
  - 确保代码生成包含测试
  - 适配 iFlow CLI 的测试框架

- [ ] **CORE-02**: Systematic Debugging 技能
  - 四阶段根因分析流程
  - 问题重现和分析
  - 最小化测试和修复

- [ ] **CORE-03**: Brainstorming 技能
  - 在编码前探索需求
  - 设计验证和方案选择
  - 避免过早实现

- [ ] **CORE-04**: Writing Plans 技能
  - 创建详细的实施计划
  - 分解为 2-5 分钟的小任务
  - 明确的验证标准

### Integration Requirements
- [ ] **INT-01**: 技能发现机制
  - 自动发现 SKILL.md 文件
  - 提取 YAML frontmatter 元数据
  - 支持技能优先级

- [ ] **INT-02**: 工具映射
  - Superpowers 工具 → iFlow CLI 工具映射
  - 子代理系统适配
  - 上下文管理适配

- [ ] **INT-03**: 钩子系统
  - SessionStart 钩子注入技能上下文
  - 技能触发机制
  - 状态管理

### Documentation Requirements
- [ ] **DOC-01**: 技术文档
  - 架构设计文档
  - 集成指南
  - API 参考文档

- [ ] **DOC-02**: 使用说明
  - 快速开始指南
  - 技能使用示例
  - 故障排除指南

- [ ] **DOC-03**: 移植说明
  - 原始技能对比
  - 适配修改说明
  - 版本兼容性

### Quality Requirements
- [ ] **QUAL-01**: 技能质量
  - 每个技能遵循 TDD 方法
  - 包含测试用例
  - 通过基线测试

- [ ] **QUAL-02**: 兼容性
  - 与 iFlow CLI v0.4.11+ 兼容
  - Windows 系统兼容
  - 跨平台路径处理

- [ ] **QUAL-03**: 可维护性
  - 清晰的代码结构
  - 完整的注释
  - 易于扩展
